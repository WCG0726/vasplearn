首页/理论基础

DFT 基本原理

密度泛函理论（DFT）是 VASP 的理论基石。

1. Hohenberg-Kohn 定理

  • 第一定理：多电子体系的基态性质由电子密度 n(r) 唯一确定
  • 第二定理：基态能量可通过能量泛函 E[n] 对密度的变分得到
核心思想：将求解 3N 维波函数的问题简化为求解 3 维电子密度，大幅降低计算复杂度。

2. Kohn-Sham 方程

公式
⎡ -∇²/2 + V_eff(r) ⎤ φᵢ(r) = εᵢ φᵢ(r)
⎣                   ⎦

V_eff(r) = V_ext(r) + V_H(r) + V_xc(r)

- V_ext(r) ：外势（原子核产生的势）
- V_H(r)   ：Hartree 势（经典库仑作用）
- V_xc(r)  ：交换关联势（量子效应）

3. 自洽场（SCF）迭代

猜测初始电子密度 n(r)
构建有效势 V_eff(r)
求解 Kohn-Sham 方程 → φᵢ, εᵢ
计算新密度 n_new(r)
判断收敛 |n_new - n_old| < ε ?
输出基态能量和电子结构

首页/理论基础/赝势与基组

赝势与平面波基组

赝势方法将芯电子"冻结"，只处理价电子，大幅降低计算成本。VASP 使用 PAW（投影缀加波）方法。

PAW 优势：比超软赝势更精确，比全电子计算更高效，是目前最主流的方法。

POTCAR 中的重要信息

POTCAR
TITEL  = PAW_PBE Si 05Jan2001   ; 赝势类型和元素
ENMAX  =   249.570              ; 推荐截断能
ZVAL   =     4.000              ; 价电子数
; Si: 3s² 3p² → 4个价电子

赝势选择指南

类型适用场景说明
PAW_PBE常规计算（最常用）GGA-PBE 泛函
PAW_LDALDA 泛函较少使用
_sv 后缀半芯态重要的体系如 3d 过渡金属
_GW 后缀GW 计算精度最高
_d 后缀含 d 电子的 p 区元素如 Ga, Ge, As
建议：初学者统一使用 PAW_PBE 赝势。对于过渡金属，考虑使用 _sv 版本以获得更精确的结果。
首页/理论基础/K 点采样

K 点采样原理

在周期性体系中，电子波函数满足布洛赫定理，需要在倒空间（k 空间）进行积分。

⟨O⟩ = (1/Nₖ) × Σ wₖ × O(kₖ)
离散 k 点求和代替布里渊区积分
物理意义：更多的 k 点意味着对布里渊区更精细的采样，但计算成本也线性增加。

K 点密度经验公式

经验法则：N_k × a ≈ 30（a 为晶格常数，单位 Å）
例如 a = 5.43 Å → N_k ≈ 30/5.43 ≈ 6 → 使用 6×6×6 网格

from pymatgen.core import Structure

    min_slab_size=10, min_vacuum_size=15,

5. 表面能计算

γ = (E_slab - N × E_bulk) / (2 × A)
γ: 表面能, A: 表面积, 2: 两个表面
python
E_slab = -156.234;  E_bulk = -3.720
N = 48;  A = 86.52
gamma = (E_slab - N * E_bulk) / (2 * A)
print(f"表面能: {gamma*16.02:.2f} J/m²")

吸附能计算

1. 定义

E_ads = E_(slab+ads) - E_slab - E_adsorbate
E_ads < 0 表示吸附放热（稳定）

2. 计算流程

优化干净表面 (E_slab)
优化自由分子 (E_mol)
构建吸附构型
优化吸附体系 (E_total)
E_ads = E_total - E_slab - E_mol

5. 自动化脚本

bashadsorption_workflow.sh
#!/bin/bash
echo "=== 干净表面 ==="
cd clean_surface; mpirun -np 16 vasp_std
E_slab=$(grep "free  energy" OUTCAR | tail -1 | awk '{print $5}')
cd ..

echo "=== 自由分子 ==="
cd free_molecule; mpirun -np 4 vasp_std
E_mol=$(grep "free  energy" OUTCAR | tail -1 | awk '{print $5}')
cd ..

echo "=== 吸附体系 ==="
cd adsorption; mpirun -np 16 vasp_std
E_total=$(grep "free  energy" OUTCAR | tail -1 | awk '{print $5}')
cd ..

E_ads=$(echo "${E_total} - ${E_slab} - ${E_mol}" | bc -l)
echo "E_ads = ${E_ads} eV"
注意：三个计算必须使用相同的 ENCUT 和 KPOINTS；自由分子放在大盒子中（20×20×20 Å），k 点用 1×1×1。

# Ti + O2 → TiO2

print(f"形成能: {delta:.3f} eV/f.u.")

首页/计算任务/缺陷计算

点缺陷形成能计算

1. 定义

E_f = E_defect - E_perfect + Σ nᵢμᵢ + q(E_VBM + E_Fermi) + E_corr
nᵢ: 原子数变化 | μᵢ: 化学势 | q: 电荷 | E_corr: 有限尺寸校正

2. 计算流程

构建完美超胞（如 3×3×3）
优化完美超胞 → E_perfect
引入缺陷（删除/替换/插入原子）
优化缺陷超胞 → E_defect
有限尺寸校正（FNV）
计算缺陷形成能

声子计算

声子谱反映晶格动力学稳定性，虚频意味着结构不稳定。

phonopy + VASP 工作流

优化结构（CONTCAR）
phonopy --vasp -d --dim="3 3 3"
生成超胞 POSCAR（DISP-001, 002...）
对每个超胞做静态计算
phonopy -f DISP-*/vasprun.xml
phonopy-bandplot → 声子谱

优化 INCAR（高精度要求）

INCAR声子前结构优化
PREC     = Accurate
ENCUT    = 500           ; 需要较高精度
EDIFF    = 1E-8          ; 严格收敛
EDIFFG   = -1E-4         ; 力收敛到 0.0001
IBRION   = 2
NSW      = 200
ISIF     = 3
ISMEAR   = 0

批量计算脚本

bash
# 生成位移超胞
cp CONTCAR POSCAR-orig
phonopy --vasp -d --dim="3 3 3"

# 批量计算
for d in DISP-*; do
    cd $d; cp ../INCAR_static ../POTCAR ../KPOINTS ./
    mpirun -np 16 vasp_std; cd ..
done

# 提取力常数并绘图
phonopy -f DISP-*/vasprun.xml
phonopy --dim="3 3 3" -c POSCAR-orig band.conf
phonopy-bandplot

#!/bin/bash
for encut in 200 250 300 350 400 450 500; do
    mkdir ENCUT_${encut}

SCF 不收敛问题

SCF 不收敛是最高频的问题。以下是完整排查清单。

🔴问题 1：电子步达到 NELM 上限仍未收敛高频▼

症状

DAV: 64 -0.275E+03 0.217E+00 -0.109E+00
（能量差持续振荡不下降）

原因

  • 初始磁矩设置不当（最常见）
  • 展宽方法错误
  • 截断能太低
  • 初始结构不合理
  • 强关联体系需要 +U

解决方案

INCAR
# 方案 1：降低收敛标准
EDIFF    = 1E-4

# 方案 2：更稳定的混合方法
AMIX     = ; 默认 0.4
BMIX class="nu">0.1           <     = 0.0001
AMIX_MAG = 0.2           ; 磁性体系
BMIX_MAG = 0.0001

# 方案 3：增加最大电子步
NELM     = 200

# 方案 4：Kerker 预处理
IMIX     = 'Kerker'
MAXMIX   = 40

# 方案 5：金属用正确展宽
ISMEAR   = 1
SIGMA    = 0.2
排查顺序：① MAGMOM 设置 ② ISMEAR ③ 降低 AMIX ④ 检查原子距离 ⑤ 是否需要 +U
🔴问题 2：DENTET - 能带数不足高频▼

症状

WARNING: DENTET: can't reach specified NBANDS
NELECT = 120.0000 NBANDS = 64

解决方案

INCAR
NBANDS   = 160
# 经验: NBANDS ≥ NELECT/2 + N_atoms/2
🟡问题 3：ZBRENT - 线搜索失败常见▼

症状

ZBRENT: fatal error in bracketing

解决方案

INCAR
IBRION   = 1             ; CG → RMM-DIIS
POTIM    = 0.01          ; 缩小步长
🟡问题 4：SCF 收敛但结果不合理常见▼
检查项方法
POTCAR 泛函grep TITEL POTCAR
ENCUT 足够收敛性测试
K 点足够收敛性测试
自旋极化含 3d/4f 金属通常需要
是否需要 +U强关联体系
🔴问题 5：VERY BAD NEWS! SGRCON 错误高频▼

症状

VERY BAD NEWS! internal error in subroutine SGRCON
The product of the three lattice vectors is negative

解决方案

晶格矢量构成的右手系方向错误。检查 POSCAR 中三个晶格矢量的顺序，交换其中两行使其构成右手坐标系。或用 VESTA/pymatgen 重新生成 POSCAR。

首页/错误诊断/离子步问题

离子步问题诊断

🔴问题 1：达到 NSW 上限仍未收敛高频▼

解决方案

  • 增大 NSW = 300
  • 放宽 EDIFFG = -0.05
  • 换优化方法 IBRION = 1
  • 减小步长 POTIM = 0.2
🔴问题 2：原子跑飞 / 结构崩塌高频▼

原因

  • 初始结构原子距离太近
  • POTIM 步长太大
  • POSCAR 坐标类型写错
  • POSCAR 和 POTCAR 元素顺序不一致

解决方案

INCAR
POTIM    = 0.1           ; 缩小步长
EDIFF    = 1E-4
EDIFFG   = -0.1
🟡问题 3：力不收敛但能量收敛常见▼
  • 降低 EDIFF = 1E-7
  • 增加 ENCUT
  • 加密 K 点
  • 使用 IBRION = 1
🟡问题 4：ISIF=3 时晶格参数异常常见▼
可能原因解决
GGA-PBE 本身高估晶格正常误差 ~1-2%
需要杂化泛函使用 HSE06
需要 +U添加 LDAU 参数

内存与性能问题

🔴问题 1：Out of memory / Killed高频▼
forrtl: severe (41): insufficient virtual memory
或直接显示 Killed
INCAR
LREAL    = Auto          ; 实空间积分（大体系）
NSIM     = 1             ; 减少同时处理的轨道数
内存估算：每个原子约需 50-200 MB。100 原子、ENCUT=500 约需 20-40 GB 总内存。
🟡问题 2：计算速度太慢常见▼
INCAR性能优化
# 实空间投影（>50 原子）
LREAL    = Auto

# 电子步加速
ALGO     = Fast
MAXMIX   = 40

# I/O 优化
LWAVE    = .FALSE.
LCHARG   = .FALSE.
NWRITE   = 0
参数说明建议
NPAR能带并行数设为节点数
KPARK 点并行数整除 K 点数
NCORE每组核心数每节点核心数
INCAR并行设置（64 核）
NPAR     = 8
KPAR     = 2
NCORE    = 8
🟡问题 3：磁盘空间不足常见▼
forrtl: severe (30): open unit, error on file OPEN
  • 检查磁盘：df -h
  • 删除 WAVECAR（最大文件）
  • 设置 LWAVE = .FALSE.

bash
grep TITEL POTCAR       # POTCAR 元素
head -6 POSCAR           # POSCAR 元素

🔴问题 2：POSCAR 格式错误高频▼
错误说明
缩放因子为 0第二行不能为 0
坐标类型拼错必须是 Direct 或 Cartesian
原子数与坐标不匹配检查总数
多余空行文件末尾不要有空行
Windows 换行符dos2unix POSCAR
🟡问题 3：KPOINTS 格式错误常见▼
  • 第 2 行不是 0
  • Gamma 拼写错误
  • Line-mode 缺少空行分隔线段

🟡问题 2：力很大（> 1 eV/Å）常见▼
bash
# 查看最大力
grep "TOTAL-FORCE" OUTCAR -A 20 | tail -20

# 检查原子距离
python3 -c "
from ase.io import read
atoms = read('POSCAR')
print(atoms.get_all_distances().min())
"
正常值：优化收敛后力应 < 0.01 eV/Å。如果力 > 1，说明结构有问题。
